Required Technical Specifications & Implementations:

2. Global State & State Branching Context (src/context/LeagueStateContext.tsx)
   - Implement a React Context Provider tracking the source-of-truth league state matrix.
   - Include a `branchSession()` method. This must create a deep clone of the active state and assign it a temporary session ID, allowing users to drop into a separate sandbox mode to experiment with mock moves without corrupting live data.
   - Include automated logic to generate clean draft matrices (handling accurate pick-flipping algorithms for Snake and 3RR formats) and structural head-to-head matchup grids based on settings.

3. Atomic Transaction & Operation Utility (src/utils/transactionHandler.ts)
   - Implement an atomic transaction runner (`executeTransaction`) capable of processing multi-team, multi-asset updates simultaneously. If any position cap, roster limit, or financial ceiling is violated by any party, the entire batch must roll back cleanly.
   - Implement a specialized drop router: If a player is dropped, check `acquiredTimestamp`. If dropped within the same billing day it was acquired, route the asset to the `FREE_AGENT` pool. If held past midnight, provision a dynamic `WAIVER_COLUMN` column array in the workspace and route the asset there.
   - Implement Selection vs Move state mutations: 
     * `setPlayerSelection(playerId, teamId)`: Updates `activeSelectionByTeamId`.
     * `clearPlayerSelection(playerId)`: Resets selection to null. If no move was made, player traces are untouched.
     * `executeMoveTransaction(playerId, targetSlot, teamId)`: Updates the player's transaction state. Sets `lastMovedByTeamId = teamId`, sets `isMostRecentMoveForTeam = true`. Automatically sweeps all other players previously moved by this `teamId` and sets their `isMostRecentMoveForTeam = false`, converting them to the historical translucent color tier.

4. Valuation & Asset-Isolated History Core (src/utils/valuationEngine.ts)
   - Build a Rest-of-Season (ROS) calculation method that reads weekly stats against custom scoring criteria, processing adjustments for risk profiles and coaching schemes.
   - Build a localized undo handler (`undoLastPlayerMove(playerId)`): Pops the top history frame off that single player's `playerHistoryStack` and snaps them back to their immediate previous location, preventing multi-user timeline conflicts.
